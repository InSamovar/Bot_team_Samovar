# Chef Planning Bot Prototype

Мини-прототип логики для Telegram-бота шеф-повара.

Сейчас реализовано:

- два блюда: куриный суп и запеканка;
- выбор объема приготовления: полный, на 25% меньше, на 50% меньше, на 75% меньше;
- пересчет ингредиентов по выбранному объему;
- редактирование плана перед подтверждением;
- редактирование количества продуктов в плане: нажать на цифру рядом с продуктом и ввести новое значение;
- проверка продуктов по статусам: проверить / есть / купить;
- сбор списка покупок только из позиций со статусом `Купить`;
- редактирование и подтверждение списка покупок;
- дата закупки в сохраненном списке покупок;
- сохранение подтвержденного плана на утро и списка покупок в папку `data`;
- локальный прототип Telegram Mini App в папке `webapp`;
- консольный сценарий, который имитирует будущий диалог в Telegram.

## Запуск

```bash
python3 prototype_cli.py
```

## Запуск Telegram-бота

1. Установить зависимости:

```bash
python3 -m pip install -r requirements.txt
```

2. Создать файл `.env` рядом с `.env.example` и добавить туда токен:

```bash
BOT_TOKEN=your_token_here
```

3. Запустить бота:

```bash
python3 bot.py
```

Команды бота:

- `/start` - показать меню команд;
- `/plan` - собрать новый план приготовления;
- `/morning_plan` - посмотреть подтвержденный план на утро;
- `/shopping_list` - посмотреть подтвержденный список покупок.

Если в `.env` добавить `WEBAPP_URL`, команда `/plan` покажет кнопку открытия Mini App.

## Локальная проверка Mini App

```bash
cd webapp
python3 -m http.server 8080
```

Открыть в браузере:

```text
http://localhost:8080
```

Для подключения к Telegram нужен публичный HTTPS-адрес. Для теста можно выложить папку `webapp` на Vercel/Netlify или открыть ее через туннель вроде ngrok, затем прописать адрес в `.env`:

```bash
WEBAPP_URL=https://your-url.example.com
```
